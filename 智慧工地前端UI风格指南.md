# 智慧工地大模型应用系统前端 UI 风格指南

## 1. 定位

采用“企业级智慧工地后台 + 轻量数据驾驶舱”风格。

目标不是普通 CRUD 后台，也不是纯大屏展示，而是面向项目管理人员和技术人员的 AI 工作台。页面需要突出项目隔离、任务状态、文档处理、智能分析、结果可追溯。

## 2. 视觉方向

- 主风格：企业后台管理系统。
- 视觉气质：专业、清晰、稳定，偏工程行业。
- 核心关键词：工地、项目、AI、流程、状态、审查、报告、知识库。
- 布局方式：左侧菜单 + 顶部项目切换 + 内容区卡片化。
- 信息呈现：表格、上传、任务进度、状态标签、JSON 结果、来源引用卡片。

## 3. 不采用的风格

- 不做纯大屏风：业务页面包含大量表单、表格和操作流程，纯大屏不适合日常使用。
- 不做极简白板风：系统需要表达任务状态、文档进度、审查结果和 AI 过程。
- 不做暗黑优先：企业和政府类验收场景更适合浅色办公界面。
- 不做花哨 AI 聊天产品风：系统不只是问答，还包括合规审查、报告管理、知识库和 OCR。

## 4. 色彩规范

建议使用浅色主题，蓝灰为基础，青绿和工程橙作为业务强调色。

| 用途 | 颜色 | 说明 |
| --- | --- | --- |
| 主色 | `#1E5EFF` | 工业蓝，用于主按钮、导航选中、关键链接 |
| 成功/知识库/AI 结果 | `#0F766E` | 青绿色，用于成功状态、知识库、AI 输出 |
| 处理中/警告/重点任务 | `#F59E0B` | 工程橙，用于处理中、警告、待处理任务 |
| 错误/严重问题 | `#EF4444` | 红色，用于失败、错误、严重问题 |
| 正文/辅助信息 | `#64748B` | 蓝灰色，用于正文、次级信息、表格辅助字段 |
| 页面背景 | `#F8FAFC` | 浅灰背景，用于整体页面底色 |

## 5. 布局规范

- 使用左侧菜单承载主要模块导航。
- 顶部栏固定展示当前项目、项目切换、用户信息和退出登录。
- 内容区采用卡片化分区，避免整页平铺信息。
- 页面首屏应明确告诉用户当前项目、当前模块、当前状态。
- 表格和表单保持企业后台常规信息密度，不要过松。
- 桌面端为主，关键操作在常见分辨率下不能遮挡或横向溢出。

## 6. 页面设计建议

### 6.1 登录页

- 使用工地线框、建筑轮廓、网格或蓝灰渐变背景。
- 登录卡片保持简洁。
- 标题突出“智慧工地大模型应用系统”。
- 登录失败必须给出明确提示。

### 6.2 首页工作台

首页做成轻量驾驶舱，建议展示：

- 当前项目。
- 知识库数量。
- 待处理任务。
- 最近报告。
- 最近审查。
- OCR 处理状态。
- 快捷入口：知识问答、合规审查、报告生成、OCR 识别、知识库管理。

### 6.3 知识问答页

- 左侧：会话列表。
- 中间：对话区。
- 右侧：来源引用、数据库结果、相关文档。
- 答案卡片支持来源标号和反馈按钮。
- 接口请求中必须有 loading 状态。

### 6.4 合规审查页

- 上方：模板选择、文档上传、发起审查。
- 中间：审查进度。
- 下方：问题列表、问题定位、修改建议、JSON 结果。
- 问题严重程度使用状态色区分。
- 审查结果支持复制或导出。

### 6.5 报告管理页

- 列表页使用标准表格，支持筛选、分页和状态展示。
- 详情页突出版本、状态、预览、下载、重新生成。
- 报告生成任务必须展示进度或阶段日志。
- 下载 Word/PDF 时处理 Blob 异常。

### 6.6 OCR 识别页

- 左侧：文件上传和图片/文档预览。
- 右侧：识别字段表单。
- 底部：JSON 结果。
- 字段置信度使用颜色标识。
- 支持人工修订、保存、复制 JSON、下载结果。

### 6.7 知识库管理页

- 使用知识库卡片 + 文档表格组合。
- 文档表格展示解析状态、入库状态、失败原因和操作。
- 支持上传、触发入库、启停用、删除。
- 数据必须随当前项目切换刷新。

## 7. 通用组件要求

- 文件上传组件：支持类型校验、大小校验、上传状态和失败提示。
- 分页表格组件：支持 loading、分页、空状态和错误提示。
- 搜索筛选组件：支持重置和查询。
- 弹窗表单组件：支持必填校验、提交 loading 和失败提示。
- 状态标签组件：统一处理成功、处理中、失败、禁用等状态。
- 进度条组件：用于报告生成、OCR、知识库入库等长任务。
- JSON 查看器：支持格式化展示、复制和下载。
- 下载按钮：统一处理 Blob 下载和错误响应。

## 8. 交互规则

- 所有页面必须处理 loading、empty、error。
- 所有接口失败必须展示明确提示，不直接暴露后端堆栈。
- 401 跳转登录页。
- 403 跳转无权限页。
- 无权限菜单和按钮不展示。
- 文件上传、报告生成、OCR 识别、知识库入库等耗时操作必须展示状态、进度或阶段日志。
- AI 结果必须尽量展示来源、置信度、JSON 或可追溯信息。
- 优先保证功能闭环，再优化视觉细节。

## 9. Codex 编码约束

后续让 Codex 写前端代码时，遵循以下规则：

- 前端技术栈使用 Vue 3 + TypeScript + Vite + Pinia + Vue Router + Axios + Element Plus。
- 前端目录使用 `frontend/`。
- API 基础地址从 `.env` 读取：`VITE_API_BASE_URL`。
- 所有接口通过 `src/utils/request.ts` 统一封装。
- 不让前端直接调用 Python 服务、数据库、MinIO 或向量库。
- 通用上传、分页、状态、JSON 展示等能力必须做成可复用组件。
- 页面风格遵循本文档，不做纯大屏风、不做暗黑优先、不做花哨聊天产品风。
